# Engenharia de Requisitos - Cartografia Participativa Quilombola

Este documento apresenta a especificação de Requisitos Funcionais (RF) e Não Funcionais (RNF) para o sistema de Cartografia Participativa da comunidade quilombola de Itapecuru-Mirim, com foco na preservação da memória e alta usabilidade.

## Tabela de Requisitos

| ID | Tipo (RF/RNF) | Nome | Descrição |
| :--- | :--- | :--- | :--- |
| **RF01** | RF | Marcação de Pontos Culturais e Históricos | O sistema deve permitir que os moradores marquem geograficamente no mapa os locais sagrados, áreas de festas tradicionais, pontos de plantio e caminhos antigos. |
| **RF02** | RF | Registo de Relatos Orais em Áudio | O sistema deve permitir a gravação ou o upload de arquivos de áudio diretamente através do telemóvel/celular para associar as histórias contadas pelos anciãos aos pontos do mapa. |
| **RF03** | RF | Upload de Acervo Fotográfico e Histórico | O sistema deve disponibilizar uma funcionalidade para carregar fotos e imagens de documentos antigos, vinculando-os ao ponto geográfico correspondente. |
| **RF04** | RF | Linha do Tempo Integrada | O sistema deve oferecer uma visualização cronológica (linha do tempo) associada aos pontos mapeados para organizar os acontecimentos históricos da comunidade. |
| **RF05** | RF | Consulta e Navegação Pública | O sistema deve permitir que qualquer membro da comunidade navegue pelo mapa interativo e aceda livremente aos áudios, fotos e textos cadastrados. |
| **RNF01** | RNF | Interface de Alta Usabilidade (Foco em Idosos) | A interface do sistema deve ser extremamente simples e intuitiva, contendo botões grandes, ícones ilustrativos e poucos passos para a execução de tarefas, garantindo o uso por anciãos com baixa literacia digital. |
| **RNF02** | RNF | Mecanismos de Acessibilidade Visual | O sistema deve possuir suporte nativo para alto contraste e controlo de tamanho de fonte, garantindo que os elementos textuais sejam legíveis para pessoas com dificuldades visuais. |
| **RNF03** | RNF | Otimização para Conexões Lentas | O sistema deve ser leve e realizar a compressão automática de imagens e áudios no upload, além de utilizar carregamento sob demanda (*lazy loading*), para operar eficientemente em redes rurais 3G/4G instáveis. |
| **RNF04** | RNF | Design Totalmente Responsivo | O sistema deve ser planeado e otimizado prioritariamente para ecrãs/telas de dispositivos móveis (smartphones), assegurando a fidelidade visual e facilidade de toque. |
