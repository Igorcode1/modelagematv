(Problema $\rightarrow$ Sistema $\rightarrow$ Software)Abaixo estão as respostas para as três perguntas fundamentais de modelagem de sistemas aplicadas aos dois projetos escolhidos:Projeto 1: Mapa Temporal Participativo de Memória QuilombolaEste projeto visa resolver o problema do registro e preservação histórica de comunidades quilombolas de forma colaborativa.1. O que entra no sistema? (Dados de Entrada / Inputs)Para que um mapa participativo temporal funcione, ele necessita receber:Dados Georreferenciados: Coordenadas (latitude e longitude) de locais históricos, antigas rotas, assentamentos ou pontos de interesse.Dados Temporais: Datas, anos ou períodos históricos associados aos eventos.Conteúdo Multimídia e Documental: Textos com relatos, arquivos de áudio (história oral), fotografias (antigas e atuais) e documentos históricos digitalizados.Dados de Usuários (Participação): Cadastros de membros da comunidade, pesquisadores ou validadores de informação que irão inserir os dados (nome, contato, credenciais).2. O que o sistema faz? (Processamento / Lógica)O comportamento interno do software com os dados recebidos:Armazenamento e Relacionamento: Vincula os arquivos multimídia e relatos às coordenadas geográficas e ao eixo de tempo correspondente no banco de dados.Gestão de Colaboração: Gerencia as submissões feitas pelos usuários (permite curadoria/aprovação do material antes de ir a público).Filtro e Busca: Processa pesquisas cruzadas (ex: buscar histórias apenas de uma década específica ou de uma região específica do mapa).3. O que ele entrega? (Saída / Outputs)O resultado final gerado para o usuário final:Interface Visual (Mapa e Linha do Tempo): Um mapa interativo onde os pontos podem ser clicados para revelar as histórias, sobreposto a uma linha do tempo (timeline) navegável.Acervo Digital: Um repositório organizado e acessível de salvaguarda da memória quilombola.Relatórios/Exportação: Possibilidade de exportar dados históricos compilados para pesquisas acadêmicas ou projetos educacionais.Projeto 2: EDA / Permanência EstudantilEste projeto visa resolver o problema da evasão escolar, utilizando dados (provavelmente Análise Exploratória de Dados - EDA) para garantir que os alunos continuem seus estudos.1. O que entra no sistema? (Dados de Entrada / Inputs)Para analisar a permanência, o sistema precisa ser alimentado com:Dados Acadêmicos: Notas das avaliações, frequência (faltas/presenças) e histórico de reprovações anteriores.Dados Socioeconômicos: Informações do cadastro do aluno (renda familiar, distância da residência até o campus, dependência de transporte escolar, auxílios recebidos).Dados de Engajamento: Registros de acessos a sistemas acadêmicos (como Moodle/Q-Acadêmico), retiradas de livros na biblioteca e atendimentos psicopedagógicos.2. O que o sistema faz? (Processamento / Lógica)Como o sistema manipula essas informações:Cruzamento de Dados (EDA): Analisa padrões (ex: relaciona o aumento de faltas com a queda de notas ou com o local de moradia).Cálculo de Risco: Aplica algoritmos ou regras lógicas para identificar alunos que apresentam comportamento típico de pré-evasão (ex: se o aluno faltar 3 semanas seguidas, o status de risco aumenta).Geração de Alertas: Dispara notificações automáticas quando métricas preocupantes são atingidas.3. O que ele entrega? (Saída / Outputs)A informação útil gerada para os gestores, professores e equipe pedagógica do IFMA:Painel de Monitoramento (Dashboard): Telas com gráficos mostrando o índice geral de permanência/evasão do campus por turma ou curso.Listas de Alerta Precoce: Relatórios destacando nominalmente os alunos que precisam de intervenção pedagógica ou assistência social imediata.Métricas de Efetividade: Dados que mostram se as ações de assistência estudantil estão realmente ajudando a manter os alunos na escola.
